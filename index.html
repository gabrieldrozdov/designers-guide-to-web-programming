Unit 1

Typography

The designer’s most important material is written language, aka typography. Typography on the web differs wildly from typography for other mediums, like books. On the web, we can’t place type wherever we want it to go. Instead, we have to define a set of rules that pieces of typography, called HTML elements, will follow.

In this unit, we’ll learn about the basics of HTML and CSS by working with typography. We’ll start from nothing and make an empty website (literally) say something.

Coding Resources

In this guide, we’ll frequently make use of a few online resources. Here’s a little background on what they are.

From GD with GD (the creator of this site), we have two other websites: Baby Steps and DEMOLAND. Baby Steps is a guide that’ll help you start coding and launch your first website from scratch. DEMOLAND will teach you how to code through a collection of demos, presented entirely in your web browser.

Then comes the two main repositories of web documentation: W3Schools and MDN Web Docs. These sites provide extensive information about EVERYTHING related to web programming. W3Schools has a friendlier (and sometimes simplified) approach, while MDN goes fully in-depth. That’s because MDN is run by some of the people that make the web run at all, namely Mozilla (developers of Firefox).

Lastly, I’ve linked to CodePen and CSS Tricks because they’re helpful resources to solve specific coding problems. We won’t be referencing them much, but they’re still good to know!

Styling Typography

There’s not much you can do with just HTML. To start making your website pretty, we’ll need to use CSS. CSS (Cascading Style Sheets) is another programming language that works with HTML to style HTML elements.

CSS mainly uses two things: properties and classes. Properties define the different visual characteristics of HTML elements, while classes organize sets of properties so that you can apply all of them at once wherever you want. We can even embed CSS right into an HTML element’s opening tag!

Unit 2

The Box Model

A well-known secret of the Internet is that everything is technically a box. Even though we have different HTML elements, each one is essentially a preset style for the same boxy content. Learning how those boxes work is the key to creating a great, easy-to-use website.

In this unit, we’ll dive deep into the so-called “box model” that governs HTML elements and learn how to start laying out our pages.

Basics of the Box Model

What does it mean that every HTML element is a box? It means that every HTML shares similar properties about how it gets arranged on a page. Elements are just rectangles in other rectangles, and we can use CSS to control the size and positioning of these shapes.

Unit 6

Data

One way JavaScript is so different from HTML and CSS is that it can actually write HTML and CSS for you. Once you realize that, you might ask yourself: how much of my website can I code just using JavaScript? The answer is quite a lot, but it helps to have some data to pull from.

In this final unit, we’ll look at how we can use JavaScript and data to create complex websites without having to write much HTML or CSS by hand.
